CLI reference

Scotia is controlled through a single binary. All commands accept --log-root and --git-commit global flags.

scotia run

Wrap an agent process and capture its telemetry.

scotia run --agent claude-code --task "refactor auth" -- claude

scotia list

List stored run artifacts.

scotia list

scotia validate

Check a stored run for structural issues.

scotia validate scotia-log/2026-07-03/run_*.json

scotia diff

Compare two runs by actions, model routes, and errors.

scotia diff run_a.json run_b.json

scotia regression

Generate deterministic assertions from a captured run.

scotia regression run_a.json

scotia summary

Print the Markdown synthesis for a run.

scotia summary run_a.json

Event algebra

The canonical schema contains nine event types. Together they reconstruct an agent execution without storing full transcripts.

  • run_started — agent process began
  • prompt_submitted — user or system input
  • action_invoked — tool or shell call
  • action_result — tool or shell result
  • model_routed — planner/executor routing decision
  • response_chunk — agent output
  • error_or_retry — failure or retry signal
  • state_delta — file or environment change
  • run_finished — agent process ended

Operations include validate, action_graph, diff_runs, and regression_suite.

Interceptors

Each supported agent has a dedicated interceptor in src/interceptors/. Interceptors are stateful parsers that turn agent-specific output patterns into canonical events.

They handle tool prefixes, JSON payloads, bracketed actions, routing hints, unified diff blocks, retry/error signals, and stderr classification.

Testing strategy

Scotia ships with 149+ tests across unit, integration, property-based, concurrency, and end-to-end categories.

cargo test        # run everything
cargo build --release

Stress suites live in tests/stress_*.rs and cover large inputs, concurrent writes, CLI round-trips, and git commit round-trips.
